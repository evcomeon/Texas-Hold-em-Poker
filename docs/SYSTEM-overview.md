# Poker Game System Overview

本文件是整个 `poker-game` 系统的快速说明文档，目标是让开发者、AI 模型或代理在最短时间内理解系统边界、核心路径、重要文件和当前风险。

## 1. 系统定位

这是一个在线德州扑克平台原型，不只是单机牌局演示。

系统包含：
- 用户认证
- 多人实时现金桌
- 观战与补位
- 排行榜
- 钱包绑定
- 链上充值
- 外挂客户端与自动化脚本
- 锦标赛外挂编排基础

当前核心玩法仍是"现金桌"，不是原生比赛服。

## 2. 顶层目录

- `client/`
  - 前端页面与交互
- `server/`
  - 主后端，提供 REST + Socket.IO + 游戏逻辑
- `recharge-service/`
  - 独立充值监听服务
- `clients/openclaw/`
  - 外挂客户端、机器人、锦标赛脚本、后续压测脚本应主要放这里
- `docs/`
  - 项目文档

### 3.1 前端

路径：
- `client/index.html`
- `client/main.js`

说明：
- 使用 Vite + 原生 JS/CSS。
- 负责登录、大厅、对局、聊天、排行榜、钱包绑定和充值入口。
- 前端不是主要业务中心，核心规则基本都在后端。

### 3.2 主后端

入口：
- `server/index.js`

职责：
- HTTP API
- WebSocket 连接
- 认证
- 大厅与分桌
- 房间广播
- 游戏引擎驱动
- 用户、排行榜、钱包、充值相关接口

### 3.3 游戏引擎

关键文件：
- `server/game/engine.js`
- `server/game/evaluator.js`
- `server/game/deck.js`

职责：
- 发牌
- 盲注
- 下注轮转
- side pot
- 摊牌
- winner 结算
- ready next hand 逻辑

说明：
- 牌局规则最集中、也最不应该被比赛需求直接污染。
- 除非确认 bug，否则不要随意改规则层。

### 3.4 大厅与房间

关键文件：
- `server/socket.js`
- `server/lobby.js`

职责：
- Socket 鉴权
- 排队
- 分桌
- 观战
- 补位
- 广播
- 断线处理

说明：
- 这里是当前系统最容易出现联机问题的地方。
- 近期已修复为"单实例可并行多桌"，但整体仍是现金桌大厅模型。

### 3.5 数据库

关键文件：
- `server/db/schema.js`
- `server/models/`

核心表：
- `users`
- `game_records`
- `game_participants`
- `transactions`
- `user_wallets`
- `recharge_orders`
- `api_keys`
- `wallet_bind_nonces`

外挂/比赛附加表：
- `tournaments`
- `tournament_rounds`
- `tournament_players`
- `tournament_tables`
- `tournament_table_players`

说明：
- 业务数据和调试数据还没有完全分层。
- 后续建议新增动作日志表，不要继续把排障信息硬塞进现有业务表。

### 3.6 独立充值服务

入口：
- `recharge-service/src/index.js`

职责：
- 监听链上 ERC-20 转账
- 匹配充值订单
- 更新到账状态
- 给用户增加筹码

说明：
- 这是独立进程，不应和游戏实时逻辑混在一起。
- 最近已修复过数据库写表错误，但仍需要完整联调。

### 3.7 外挂客户端与脚本

目录：
- `clients/openclaw/`

用途：
- AI 客户端接入
- 多机器人联机验证
- 锦标赛账号生成
- 锦标赛执行
- 后续压测脚本

当前已有：
- `index.js`
- `test-multi-bot.js`
- `create-tournament-accounts.js`
- `run-tournament.js`
- `tournament-db.js`

说明：
- 新增自动化能力优先放这里。
- 这样能减少对游戏服务本体的侵入。

## 6. 当前已知的重要事实

1. 当前游戏服务器是现金桌模型，不是原生比赛模型。
2. 单实例现在已经支持同盲注级别并行开多桌。
3. Socket 事件名曾有过服务端和客户端不一致的问题，修过一次，后续修改要同步检查。
4. 钱包登录、API Key 鉴权、充值监控都曾有过明显 bug，已做过第一轮修复。
5. 当前最缺的是压测、日志、恢复能力，不是新功能数量。

## 7. 最近修复过的关键问题

### 7.1 钱包登录 token 生成错误

影响文件：
- `server/routes/wallet.js`
- `server/auth.js`

结果：
- 钱包登录路径现在正确使用 `generateJWT`。

### 7.2 WebSocket API Key 不可用

影响文件：
- `server/socket.js`

结果：
- 现在支持 `socket.handshake.auth.apiKey`，并校验 `game` 或 `write` 权限。

### 7.3 充值监控写错表

影响文件：
- `recharge-service/src/services/blockchain.js`

结果：
- 充值交易幂等和到账记录改为落在正确交易表，不再把交易字段写进订单表。

### 7.4 客户端事件名不一致

影响文件：
- `clients/openclaw/index.js`
- `clients/openclaw/test-multi-bot.js`

结果：
- 已对齐当前服务端事件名。

### 7.5 单实例多桌问题

影响文件：
- `server/lobby.js`

旧行为：
- 开出一桌后，剩余排队用户会被大量塞成该桌观战者。

当前行为：
- 优先填等待中的空桌。
- 队列够人数时持续新开桌。
- 只有不足以再开新桌的零散用户才进入观战。

### 7.6 观战者匹配问题

影响文件：
- `server/lobby.js`
- `server/game/engine.js`
- `server/socket.js`

结果：
- 输光筹码的玩家正确转为观战者。
- 观战者筹码不足不参与匹配。
- 下一局玩家和观战者正确匹配。
- 支持11个bot同时测试。

## 8. 关键风险

### 8.1 内存态过重

风险位置：
- `server/lobby.js`
- `server/game/engine.js`

问题：
- 房间和牌局主要存在内存。
- 服务重启后难以恢复。
- 多实例扩容困难。

### 8.2 Socket 层过重

风险位置：
- `server/socket.js`

问题：
- 同时承担鉴权、lobby、game、广播、聊天、重连。
- 后续改动容易产生回归。

### 8.3 日志不足

问题：
- 缺少结构化日志和动作级日志。
- 压测后定位 bug 成本高。

### 8.4 断线语义不统一

问题：
- 有时移除玩家，有时标记 disconnected。
- 对轮次、补位、结束条件有潜在影响。

## 9. 性能指标与容量规划

### 9.1 当前性能指标

| 指标 | 当前值 | 说明 |
|------|--------|------|
| 单房间最大玩家数 | 8 | 游戏规则限制 |
| 单房间最大观战者 | 无限制 | 内存限制 |
| 单实例最大房间数 | 未测试 | 取决于内存和 CPU |
| 单实例最大连接数 | 未测试 | 取决于 Socket.IO 配置 |
| 单手牌平均时长 | 2-5 分钟 | 取决于玩家决策速度 |
| 准备超时时间 | 30 秒 | 可配置 |
| 动作超时时间 | 30 秒 | 可配置 |

### 9.2 容量规划建议

| 场景 | 预估连接数 | 预估房间数 | 建议配置 |
|------|-----------|-----------|----------|
| 开发测试 | 10-20 | 1-2 | 单实例，2核4G |
| 小规模运营 | 100-200 | 10-20 | 单实例，4核8G |
| 中规模运营 | 500-1000 | 50-100 | 多实例 + 负载均衡 |
| 大规模运营 | 5000+ | 500+ | 多实例 + Redis + 数据库分片 |

### 9.3 性能瓶颈点

1. **内存中的房间状态** - 每个房间占用内存，房间数受限于内存大小
2. **Socket.IO 单实例** - 连接数受限于单进程文件描述符限制
3. **数据库写入** - 每手牌结算时的数据库写入可能成为瓶颈
4. **广播消息** - 大量观战者时广播消息可能成为瓶颈

| 性能 | 内存使用 | 进程内存占用 |
| 性能 | CPU 使用 | 进程 CPU 占用 |
| 数据库 | 查询延迟 | 数据库响应时间 |
| 数据库 | 连接池状态 | 连接池使用情况 |

### 10.3 建议添加的告警规则

| 告警 | 条件 | 级别 |
|------|------|------|

| 内存使用过高 | > 80% 可用内存 | Warning |
